QUICKEST WAY TO ADD DATA
================================================================================

If you have temporary Firestore rule access:

STEP 1: Update Firestore Rules (temporary)
  Go to: Firebase Console > Firestore > Rules
  Add to rules:
    match /eodReports/{document=**} {
      allow read, write: if true;
    }
  Click: Publish

STEP 2: Run the seed script
  Command: node seed-eod-reports-rest.js

Expected output:
  Starting to seed EOD Reports using REST API...
  Created EOD Report: 24/24 animals fed on Sat Jun 27 2026
  Created EOD Report: 23/24 animals fed on Fri Jun 26 2026
  ...
  Finished seeding 5 EOD Reports!

STEP 3: Restore Firestore Rules (secure)
  Go back to: Firebase Console > Firestore > Rules
  Update to secure state:
    match /eodReports/{document=**} {
      allow read, write: if request.auth.uid != null;
    }
  Click: Publish

STEP 4: Verify in Firebase Console
  Navigate to: Firestore > eodReports collection
  Should see 5 documents with dates Jun 23-27, 2026

================================================================================
TROUBLESHOOTING
================================================================================

Q: Getting "Permission Denied" error?
A: Verify Firestore rules were updated and published.
   Wait 5-10 seconds for changes to propagate.
   Try script again.

Q: Data not appearing in Firestore?
A: Check if rules were published successfully.
   Try manually checking Firebase Console.
   Check browser console for errors.

Q: Want to delete all seed data?
A: Firebase Console > Firestore > eodReports > Select all > Delete
   Or run delete script (create one if needed)

Q: Which method should I use?
A: REST API: Fastest if you can temporarily modify rules
   Admin SDK: More secure but requires setup
   Manual: Via Firebase Console if other methods fail
