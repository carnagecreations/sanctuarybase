================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

TECHNOLOGY STACK:
- Vue 3 with Composition API
- Pinia state management
- Firestore (NoSQL database)
- Firebase Authentication (existing)
- TypeScript-ready (no strict types enforced)

FIRESTORE COLLECTIONS:
1. animalMedical
   - 8 fields per document
   - Supports unlimited records per animal
   - Indexed on: animalId, date

2. vaccinations
   - 8 fields per document
   - Supports unlimited vaccinations per animal
   - Indexed on: animalId, dueDate, nextDueDate

AUTHENTICATION & AUTHORIZATION:
- Read: All authenticated users
- Write: Staff and Admin only (via isPrivileged() function)
- Follows existing SanctuaryBase security model

DATE HANDLING:
- All dates converted to/from Firestore Timestamp objects
- Frontend formatting: 'Month DD, YYYY' format
- Relative time display: '2h ago', '3d ago', etc.
- Status calculations based on current system time

FORM VALIDATION:
Medical Records:
  - Required: description
  - Optional: type, vet, date, notes, cost

Vaccinations:
  - Required: vaccine name
  - Optional: dates, vet, batch number

================================================================================
DATA STRUCTURE
================================================================================

MEDICAL RECORD DOCUMENT:
{
  animalId: "string",           // FK to animals collection
  date: Timestamp,              // When service was provided
  type: "string",               // vet visit, diagnosis, treatment, etc.
  description: "string",        // What was done
  veterinarian: "string",       // Provider name
  notes: "string",              // Additional details
  cost: number,                 // Optional cost tracking
  prescriptions: [],            // Array for future use
  createdAt: Timestamp,         // Auto-generated
  updatedAt: Timestamp          // Auto-updated
}

VACCINATION DOCUMENT:
{
  animalId: "string",           // FK to animals collection
  vaccineName: "string",        // Vaccine name (FVRCP, Rabies, etc.)
  dueDate: Timestamp,           // Original due date
  completedDate: Timestamp,     // When vaccine was given (null if pending)
  nextDueDate: Timestamp,       // When booster is due
  veterinarian: "string",       // Provider name
  batchNumber: "string",        // Lot/batch tracking
  createdAt: Timestamp,         // Auto-generated
  updatedAt: Timestamp          // Auto-updated
}

================================================================================
USAGE GUIDE
================================================================================

VIEWING MEDICAL HISTORY:
1. Go to Animals page
2. Click on an animal to open AnimalDetail
3. Click "Meds" tab to see medical records
4. Records appear in reverse chronological order (newest first)

ADDING A MEDICAL RECORD:
1. In MedsTab, fill out the "Add Medical Record" form
2. Enter required field: description
3. Optional: select type, enter vet name, notes, cost
4. Click "Add Record"
5. Record appears at top of list immediately

VIEWING VACCINATIONS:
1. In AnimalDetail, click "Vaccines" tab
2. See summary statistics at top
3. View all vaccination records with color-coded status
4. Overdue (red), Due Soon (yellow), Current (green)

ADDING A VACCINATION:
1. In VaccinesTab, fill out "Add Vaccination" form
2. Enter vaccine name
3. Select/enter dates for given and next due
4. Optional: enter vet name and batch number
5. Click "Log Vaccine"
6. Vaccine appears in list with auto-calculated status

VIEWING MEDICAL ALERTS:
1. In AnimalDetail, click "Profile" tab
2. Scroll to "Medical Alerts" section
3. See overdue vaccines and due-soon vaccines
4. Color-coded for quick scanning (red = urgent)
5. No alerts message when all vaccines current

EDITING A RECORD:
1. Find the record/vaccine to edit
2. Click "Edit" button
3. Modify fields in form
4. Changes saved automatically (MedsTab only currently)
5. For VaccinesTab, delete and re-add to modify

DELETING A RECORD:
1. Click "Delete" button on record
2. Confirm deletion in popup
3. Record removed from Firestore and UI
4. Cannot be undone

================================================================================
TROUBLESHOOTING
================================================================================

RECORDS NOT SAVING?
→ Check Firestore security rules are deployed
→ Verify user is authenticated and is staff/admin
→ Check browser console for error messages

RECORDS NOT LOADING?
→ Verify Firestore connection
→ Check user is authenticated
→ Verify animal ID is valid
→ Look for console errors

DATES SHOWING WRONG?
→ Check system timezone
→ Verify dates are Firestore Timestamps (not strings)
→ Confirm date format in formatDate() function

FORMS NOT VALIDATING?
→ Check required fields (description for records, vaccine name for vaccines)
→ Verify form inputs are properly bound with v-model
→ Look for validation logic in addMedicalRecord() and addNewVaccine()

STATUS BADGES WRONG COLOR?
→ Verify getStatus() calculation logic
→ Check date comparisons with current date
→ Ensure overdue = nextDueDate < today
→ Ensure due-soon = within 30 days
