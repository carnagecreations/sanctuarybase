SCHEMA FOR EACH RECORD
======================
- volunteerId: Unique volunteer identifier
- volunteerName: Full name of volunteer
- shiftId: Unique shift identifier
- shiftType: morning | afternoon | evening
- startTime: HH:MM format
- endTime: HH:MM format
- duration: Calculated duration string
- date: YYYY-MM-DD format
- sections: Array of 4 facility sections
  - id, name, completedAt, notes, animalsChecked, feedingCompleted
- totalAnimalsChecked: Sum of all animals in sections (14)
- issuesCount: Number of issues encountered
- issuesReported: Array of issue objects with type, severity, description
- notes: Round summary/notes
- createdAt: Firestore Timestamp
- updatedAt: Firestore Timestamp

FIRESTORE SECURITY RULES REQUIRED
==================================
For the seed script to work, add this rule:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /roundsHistory/{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
