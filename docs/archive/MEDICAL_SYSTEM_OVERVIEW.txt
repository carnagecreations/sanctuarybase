┌─ DATA FLOW EXAMPLE ────────────────────────────────────────────────────────┐
│                                                                             │
│  ADDING A MEDICAL RECORD:                                                 │
│                                                                             │
│  1. User fills MedsTab form:                                             │
│     Description: "Annual checkup"                                        │
│     Type: "examination"                                                  │
│     Vet: "Dr. Smith"                                                     │
│     Date: 2024-03-05                                                     │
│     Notes: "All healthy"                                                 │
│     Cost: 150                                                             │
│                                                                            │
│  2. Clicks "Add Record" button                                           │
│                                                                            │
│  3. Form validates required field (description)                          │
│                                                                            │
│  4. medicalRecords.addRecord() called with data                         │
│                                                                            │
│  5. Data sent to Firestore animalMedical collection                      │
│                                                                            │
│  6. Firestore creates document with:                                    │
│     - Auto-generated ID                                                 │
│     - animalId (from component prop)                                    │
│     - All form fields                                                   │
│     - createdAt timestamp                                               │
│                                                                            │
│  7. Store updates local state                                            │
│                                                                            │
│  8. VaccinesTab re-fetches data via fetchByAnimalId()                   │
│                                                                            │
│  9. Component re-renders with new record at top of list                 │
│                                                                            │
│  10. Form clears for next entry                                          │
│                                                                            │

┌─ STATUS CALCULATION LOGIC ─────────────────────────────────────────────────┐
│                                                                             │
│  VACCINE STATUS DETERMINATION:                                            │
│                                                                            │
│  const nextDue = vaccine.nextDueDate || vaccine.dueDate                  │
│  const daysUntilDue = (nextDueDate - today) / msPerDay                   │
│                                                                            │
│  IF daysUntilDue < 0                                                     │
│     → Status: "overdue" (RED BADGE)                                      │
│                                                                            │
│  ELSE IF daysUntilDue <= 30                                              │
│     → Status: "due-soon" (YELLOW BADGE)                                  │
│                                                                            │
│  ELSE                                                                     │
│     → Status: "current" (GREEN BADGE)                                    │
│                                                                            │
