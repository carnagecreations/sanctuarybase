==========================================
HOW TO ADD BITE REPORTS SAMPLE DATA
==========================================

OPTION 1: BROWSER CONSOLE (EASIEST - 30 SECONDS)
================================================

Steps:
1. Start the SB v2 app locally (npm run dev or your command)
2. Navigate to http://localhost:5173 (or your app URL)
3. Press F12 to open Developer Tools
4. Click the "Console" tab
5. Paste this code:

import { seedBiteReports } from '/src/utils/seedBiteReports.js'
seedBiteReports().then(results => console.log('✓ Created', results.length, 'reports'))

6. Press Enter
7. Wait 2-3 seconds for completion
8. Navigate to Admin Hub > Bite Reports page
9. You should see 3 pending + 2 reviewed reports

---

OPTION 2: NODE.JS ADMIN SDK (RECOMMENDED FOR CI/CD)
===================================================

Requirements:
- Node.js installed
- Firebase service account key (.firebase/serviceAccountKey.json)

Steps:
1. Download service account key:
   - Go to Firebase Console > Project Settings
   - Service Accounts tab
   - Click "Generate New Private Key"
   - Save to: .firebase/serviceAccountKey.json

2. Install Firebase Admin SDK:
   npm install firebase-admin

3. Run:
   node functions/seed-bite-reports.js

4. Output will show:
   - Started seeding
   - Created [animal name] records
   - Completed count

---

OPTION 3: FIREBASE REST API (NO SDK)
====================================

Steps:
1. Run:
   node seed-bite-reports-rest.js

2. Will create records via REST API
3. Note: Requires Firestore rules to allow writes

HOW TO DELETE
=============

If you want to remove the sample data:

Option A: Firebase Console (recommended)
1. Go to Firebase Console > biteReports collection
2. Right-click each document
3. Click Delete
4. Confirm

Option B: Code (advanced)
Create a delete script with deleteDoc() or batch delete

---

DATA FIELDS INCLUDED
====================

Each record has:
- animalId: References the animal (e.g., "animal-001")
- animalName: Display name (e.g., "Venom")
- emoji: Animal type emoji (e.g., "🐍")
- personId: Who was bitten
- victim: Victim's name
- victimRole: "Volunteer" or "Staff"
- date: Date in YYYY-MM-DD format
- description: What happened
- severity: "Minor", "Moderate", "High"
- location: Body location bitten
- status: "Pending" or "Reviewed"
- injuryDetails: Medical description
- reportedAt: Timestamp
- createdAt: Timestamp
- witnesses: Array of names
- firstAidProvided: First aid details
- medicalTreatmentRequired: Boolean
- [If Reviewed] reviewedAt, reviewedBy, reviewNotes

---

TROUBLESHOOTING
===============

"Permission denied" error?
  → Check Firestore security rules
  → Try browser console method first
  → Ensure you're logged in to Firebase

"Collection not found"?
  → Normal! The collection is auto-created
  → Just means it's empty until seeded

Nothing appears in app?
  → Refresh the page (F5)
  → Check browser console for errors
  → Try a different method

Data doesn't match?
  → Check your Firestore Console
  → Make sure records are there
  → Clear browser cache if needed
